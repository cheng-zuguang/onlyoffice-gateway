# 管理端可观测性与临时附件实现方案

## 目标

为单一管理员提供 Gateway 实例本地的运行审计日志，以及临时附件的安全管理能力；同时将管理端统一迁移到 shadcn/ui 与 Tailwind CSS 4。

## API

所有接口均位于 `/admin/api`，使用既有 Bearer 管理员 JWT。

| 方法 | 路径 | 行为 |
| --- | --- | --- |
| GET | `/attachments` | cursor 分页列出临时附件，支持 `service_id`、`limit` 与 `cursor`。 |
| GET | `/attachments/{id}` | 返回脱敏后的附件详情。 |
| GET | `/attachments/{id}/download?variant=original|latest` | 下载 Gateway 托管的附件版本。 |
| POST | `/attachments/{id}/extend-ttl` | 以 `{ "hours": 1..168 }` 延长 TTL。 |
| DELETE | `/attachments/{id}` | 仅接受 `{ "confirm": true }` 的立即删除。 |
| POST | `/attachments/cleanup` | 立即清理已过期附件，返回清理数量。 |
| GET | `/logs` | 查询当前实例的结构化审计日志。 |

`source_url` 直连附件只返回脱敏元数据，不提供下载或代理。附件记录只暴露 source/webhook host，不暴露完整 URL、查询参数或凭据。

## 存储与 TTL

存储契约新增附件目录能力，Local 与 S3 后端按创建时间倒序、以不透明 cursor 分页。TTL 调度不再只限 Local：它会对 Gateway 所有支持的存储后端调用过期清理。Direct-source 记录的清理只删除 Gateway 元数据，不会删除业务对象。

## 审计日志

审计日志采用实例本地 JSONL，每天一个文件，文件权限为仅进程用户可读写。默认留存 14 天，通过 `ADMIN_AUDIT_LOG_DIR` 和 `ADMIN_AUDIT_LOG_RETENTION_DAYS` 配置；留存范围为 1–90 天。写入同步落盘，删除、TTL 延长、手动清理及下载在审计记录写入失败时返回 503。

日志禁止记录 Authorization、JWT、请求体、文件内容和完整带 query 的 URL。多实例聚合及 Document Server 日志不在本期范围内。

## 管理端

管理端使用 shadcn 的 `components.json` 配置、Radix primitives、CVA button 变体和 Tailwind CSS 4 Vite 插件。一级导航包括服务管理、临时附件与运行日志；界面以简体中文呈现，并沿用 CSS variable 主题 token。

## 验收与测试

- Local/S3 存储契约均能列出临时附件并返回 cursor。
- 审计日志查询可得到已写事件，且 URL query 已脱敏。
- 附件列表需要管理员 JWT，匿名请求返回 401。
- 全量 Go 测试与管理端生产构建必须通过。
