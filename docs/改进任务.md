# Gateway 改进任务

## T1：保护 ONLYOFFICE callback（完成）

- 为 callback URL 发放绑定 `document_id` 的 HMAC capability。
- callback 在下载编辑文件或写入存储前验证 capability；无效请求返回 401。
- 统一 ONLYOFFICE document key 与 `document_id`，避免 callback 无法定位文档。
- 验收：未授权请求不会访问其提交的 URL；合法 callback 仍能保存编辑结果并投递 webhook。

## T2：启动配置与上传输入保护（完成）

- 启动前校验 JWT 密钥、Document Server、TTL 和存储后端配置。
- 限制 multipart 总体积，返回 413；拒绝未知 `document_type`。
- 验收：缺少 JWT 密钥时进程不启动；超限上传不写入存储；合法 word/cell/slide/pdf 上传不受影响。

## T3：浏览器和下载边界（完成）

- 转义编辑页 HTML title，安全构造 Content-Disposition 文件名。
- SDK 仅处理自身 iframe 且来自 Gateway 的 postMessage。
- 对编辑 token 添加 Referrer-Policy，并评估短期一次性编辑会话。
- 验收：恶意文件名无法注入标签/脚本；第三方 window 消息不触发 SDK 回调。

## T4：异步保存可靠性（完成）

- 已完成：清理完成的 debounce timer；下载编辑文件时校验 2xx；`source_url` 直连模式只转发 `edited_url`，不下载编辑后文件。
- 已完成：保存任务进入可配置有界队列；Webhook 重试使用指数退避并附加短 jitter；暴露保存和 webhook 指标。
- 已验证：高频 callback 只保存最后一次；错误响应不会覆盖编辑结果；未授权 callback 不会访问其提交的 URL；保存失败会反映在 `/api/v1/metrics`。

## T5：存储性能与规模（完成）

- 已完成：普通完整下载不再执行二次 Open；S3 删除按 1,000 个对象分批；本地存储按 `CLEANUP_INTERVAL` 自动清理过期文档。
- 已完成：存储接口新增 `Stat`，条件请求和非法 Range 可只取对象元信息；`source_url` 直连模式不由 Gateway 清理业务对象。
- 已验证：超过 1,000 个对象可完整删除；过期本地文档会被自动清理；S3 `Stat` 不下载对象正文。
- 运维建议：`source_url` 直连对象生命周期由业务侧存储策略管理；Gateway 只负责本地 multipart 托管文件清理。

## T6：服务注册可靠性（完成）

- 新增/更新时验证并缓存 RSA 公钥。
- 用临时文件、fsync 和 rename 原子保存服务注册表；把持久化错误返回 Admin API。
- 验收：无效 PEM 在注册/更新时被拒绝；无效更新不会覆盖已有服务；服务注册表写入为原子替换。

## T7：生产收尾与可观测性（完成）

- 收到 `SIGTERM` / `SIGINT` 时，先停止 HTTP 接入，再取消本地过期清理调度并排空已入队的 callback 保存任务。
- `/api/v1/metrics` 输出 Prometheus `HELP` / `TYPE` 元信息；保存队列满、保存失败、webhook 最终失败均可用于告警。
- `cleanup_interval` 在 YAML 中支持 `15m`、`1h` 等时长写法，并有配置加载测试覆盖。
- 文档给出业务存储 Lifecycle 模板、回调拥塞与下游故障演练步骤；`source_url` 直连对象清理仍由业务侧负责。

## TDD 执行约束

每个验收行为独立走一次 RED → GREEN → 重构：先通过 HTTP API、SDK 公共组件或存储契约写失败测试，再做仅足以转绿的实现；不批量预写测试或跨任务重构。
