## 阶段 1：部署 ONLYOFFICE Document Server

### 1.1 拉取并启动

```bash
docker pull onlyoffice/documentserver:latest

docker run -d \
  --name onlyoffice-document-server \
  -p 18000:80 \
  -v /opt/onlyoffice/DocumentServer/data:/var/www/onlyoffice/Data \
  onlyoffice/documentserver:latest
```

### 1.2 等待就绪

```bash
# 约 30-60 秒后
curl -s http://localhost:18000/welcome/ | grep "Document Server is running"
```

### 1.3 获取 JWT Secret

```bash
docker exec onlyoffice-document-server \
  cat /etc/onlyoffice/documentserver/local.json \
  | grep -A2 '"secret"'
```

记下 `inbox.string` 的值，填入 `.env` 的 `JWT_SECRET`。

## 阶段 2：部署 Gateway

### 2.1 克隆项目

```bash
cd /opt
git clone <repo-url> onlyoffice-gateway
cd onlyoffice-gateway
```

### 2.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`：

```env
# ── Gateway ──
LISTEN_ADDR=0.0.0.0:18080
DOCUMENT_SERVER_URL=http://localhost:18000
JWT_SECRET=<从 Document Server 获取的 inbox.string>
STORAGE_DIR=./data/storage
TTL_HOURS=8
WEBHOOK_MAX_RETRIES=3

# ── Admin Panel ──
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<设置强密码>
SERVICE_STORE_PATH=./data/services.json
```

> Services 不再在 YAML 中配置，全部通过管理端界面管理。

### 2.3 构建并启动

```bash
# 构建 Go binary
make build

# 构建管理端前端
make frontend-build

# 启动 Gateway
LISTEN_ADDR=0.0.0.0:18080 \
DOCUMENT_SERVER_URL=http://localhost:18000 \
JWT_SECRET=<your-secret> \
ADMIN_USERNAME=admin \
ADMIN_PASSWORD=<your-password> \
./bin/gateway
```

### 2.4 Docker Compose（一键部署）

```bash
cp .env.example .env
# 编辑 .env，设置 JWT_SECRET 和 ADMIN_PASSWORD

docker compose up -d
```

### 2.5 启动管理端

```bash
# 开发模式（Vite dev server）
make frontend-dev

# 生产模式：将 admin-ui/dist/ 部署到 Nginx / CDN
```

打开 `http://<host>:5173/admin/login`，用 `.env` 中设置的凭据登录。

### 2.6 注册业务服务

在管理端页面中：

1. 点击 **Add Service**
2. 填写 **Service ID**（唯一标识，如 `crm-service`）
3. 粘贴业务服务的 **RSA 公钥**（PEM 格式）
4. 填写 **Webhook 域名白名单**（如 `crm.example.com`, `localhost`）
5. 提交

新增的服务**立即生效**，无需重启 Gateway。

## 阶段 3：业务服务接入

### 3.1 生成 RSA 密钥对

```bash
openssl genpkey -algorithm RSA -out service-private.pem -pkeyopt rsa_keygen_bits:2048
openssl rsa -pubout -in service-private.pem -out service-public.pem
```

将公钥内容通过管理端注册到 Gateway。私钥由业务服务保管，用于签名 JWT。

### 3.2 签 JWT 并上传文档

```javascript
const jwt = require('jsonwebtoken');
const fs = require('fs');
const FormData = require('form-data');

const GATEWAY_URL = 'http://your-gateway:18080';
const PRIVATE_KEY = fs.readFileSync('service-private.pem');

const token = jwt.sign({
  service_id:  'my-service',
  webhook_url: 'https://my-app.example.com/webhook',
  external_id: 'doc-001',
  file_name:   'document.docx',
  document_type: 'word',
}, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '60s' });

const form = new FormData();
form.append('file', fs.createReadStream('document.docx'));

const resp = await fetch(`${GATEWAY_URL}/api/v1/documents`, {
  method: 'POST',
  headers: { Authorization: `Bearer ${token}`, ...form.getHeaders() },
  body: form,
});
const { document_id } = await resp.json();
```

### 3.3 生成编辑 token

```javascript
app.get('/api/edit-token', (req, res) => {
  const token = jwt.sign({
    service_id:  'my-service',
    document_id: req.query.documentId,
  }, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '30m' });
  res.json({ token });
});
```

## 阶段 4：前端集成

### 4.1 安装 SDK

```bash
npm install @zenmind/onlyoffice-editor
```

### 4.2 嵌入编辑器

```tsx
import { OnlyOfficeEditor } from "@zenmind/onlyoffice-editor";

// 1. 获取编辑 token
const { token } = await fetch(`/api/edit-token?documentId=${docId}`).then(r => r.json());

// 2. 渲染编辑器
<OnlyOfficeEditor
  documentId={docId}
  gatewayUrl="http://gateway:18080"
  token={token}
  onSaved={async (e) => {
    const resp = await fetch(`${gatewayUrl}/api/v1/documents/${e.document_id}`, {
      headers: { Authorization: `Bearer ${await getServiceJWT()}` },
    });
    const blob = await resp.blob();
    await saveToStorage(blob);
  }}
  style={{ height: "80vh" }}
/>
```

```bash
  -H 'Authorization: Bearer <token>' \
  -d '{"id":"test-svc","public_key":"<pem>","allowed_webhook_domains":["localhost"]}'

# 2. 签 JWT 并上传
# （参考阶段 3.2）

# 3. 编辑完成，下载
curl http://localhost:18080/api/v1/documents/<doc_id>
```

## 常见问题

### Q: Document Server 无法访问 Gateway 的 /download 和 /callback？

Gateway 配置 `LISTEN_ADDR=0.0.0.0:18080`（Docker 容器需通过 host 网络访问）。

### Q: 上传时提示 "invalid token"？

检查 JWT 算法是否为 RS256、私钥与管理端注册的公钥是否配对、JWT 是否过期。

### Q: Webhook 域名白名单检查失败？

检查的是 Webhook URL 的域名部分（不含 scheme 和端口）。

### Q: 文档 TTL 过期被清理？

TTL 默认 8 小时，编辑期间每次 callback（status=1）自动续期。

### Q: 多用户协同编辑？

Gateway 首次上传生成 `document.key` 并持久化，同一 `document_id` 的所有请求复用同一 key。
