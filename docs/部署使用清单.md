# ONLYOFFICE Gateway 部署使用清单

> 版本: v0.1.0 | 2026-07-06

本文档列出从零到成功使用的**全部前置条件和操作步骤**。每一步都是必须的，按顺序执行。

## 前提条件总览

| # | 条件 | 检查命令 | 必须？ |
|---|---|---|---|

| 0.2 | Go 1.22+ | `go version` | ✅（源码启动） |

## 阶段 1：部署 ONLYOFFICE Document Server

### 1.1 拉取镜像

```bash
docker pull onlyoffice/documentserver:latest
```

### 1.2 启动 Document Server

```bash
docker run -d \
  --name onlyoffice-document-server \
  -p 8080:80 \
  -v /opt/onlyoffice/DocumentServer/data:/var/www/onlyoffice/Data \
  onlyoffice/documentserver:latest
```

### 1.3 等待就绪

```bash
# 等待 30-60 秒，然后检查
curl -s http://localhost:8080/welcome/ | grep -o "Document Server is running"
# 输出: Document Server is running
```

### 1.4 获取/配置 JWT Secret

Document Server 默认开启 JWT，需要获取其 secret：

```bash
docker exec onlyoffice-document-server cat /etc/onlyoffice/documentserver/local.json | grep -A2 '"secret"'
```

输出示例：
```json
"secret": {
  "inbox": { "string": "abc123..." },
  "outbox": { "string": "abc123..." },
  "session": { "string": "abc123..." }
}
```

**记下 `inbox.string` 的值**，这就是 Gateway 需要的 `jwt_secret`。

> 如果要自定义 secret，进入容器编辑 `/etc/onlyoffice/documentserver/local.json`，然后 `supervisorctl restart all`。

## 阶段 2：部署 Gateway

### 2.1 克隆/获取项目

```bash
cd /opt  # 或其他工作目录
git clone <your-repo-url> onlyoffice-gateway
cd onlyoffice-gateway
```

### 2.2 生成业务服务的 RSA 密钥对

**重要**：每个接入的业务服务需要一对 RSA 密钥。以下在 Gateway 服务器上生成（实际应该由业务服务自行生成，这里是为配置做准备）：

```bash
# 为业务服务生成密钥（例如 crm-service）
openssl genpkey -algorithm RSA -out crm-service-private.pem -pkeyopt rsa_keygen_bits:2048
openssl rsa -pubout -in crm-service-private.pem -out crm-service-public.pem

# 查看公钥内容（之后写入 gateway.yaml）
cat crm-service-public.pem
```

**将私钥 `crm-service-private.pem` 安全地交给业务服务团队保管。Gateway 只需要公钥。**

### 2.3 配置 gateway.yaml

```bash
cp gateway.yaml.example gateway.yaml
```

编辑 `gateway.yaml`：

```yaml
# Gateway 监听地址
# - Document Server 在本机 Docker: 0.0.0.0:18080（Docker 容器需通过 host 网络访问）
# - Document Server 在云端 & Gateway 在云端: 127.0.0.1:18080 或 0.0.0.0:18080
listen_addr: "0.0.0.0:18080"

# Document Server 地址
# - 本机 Docker: http://localhost:8080（Gateway 本机进程可直接访问）
# - 云端: https://doc.your-company.com
document_server_url: "http://localhost:8080"

# JWT Secret —— 与 Document Server 的 inbox.string 一致
jwt_secret: "你的JWT_SECRET"

storage_dir: "./data/storage"
ttl_hours: 8
webhook_max_retries: 3

services:
  - id: "crm-service"
    public_key: |
      -----BEGIN PUBLIC KEY-----
      （粘贴 crm-service-public.pem 的内容到这里）
      -----END PUBLIC KEY-----
    allowed_webhook_domains:
      - "crm.your-company.com"          # 生产
      - "crm-staging.your-company.com"  # 预发布
      - "localhost"                      # 开发测试
```

### 2.4 构建并启动

**方式 A：Docker Compose（推荐，一键启动全部）**

```bash
# 1. 配置 gateway.yaml（参考 2.3 节）
#    注意：document_server_url 填 "http://document-server"（docker-compose 服务名）
#          listen_addr 填 "0.0.0.0:18080"

# 2. 自动生成 JWT Secret（或自己指定）
export JWT_SECRET=$(openssl rand -hex 32)
echo "生成的 JWT_SECRET: $JWT_SECRET"
# 这个值会同时传递给 Document Server 和 Gateway，无需手动配置

# 3. 一键启动 Gateway + Document Server
docker compose up -d

# 4. 查看日志
docker compose logs -f gateway
```

**方式 B：仅 Gateway Docker 部署（Document Server 已单独运行）**

```bash
# 1. 配置 gateway.yaml
#    document_server_url 根据实际 Document Server 地址填写

# 2. 构建镜像
docker build   --build-arg VERSION=v0.1.0   --build-arg BUILD_TIME=$(date -u +%Y-%m-%dT%H:%M:%SZ)   --build-arg COMMIT=$(git rev-parse --short HEAD 2>/dev/null || echo "unknown")   -t onlyoffice-gateway:v0.1.0 .

# 3. 启动
docker run -d   --name oo-gateway   -p 18080:18080   -v $(pwd)/gateway.yaml:/app/gateway.yaml:ro   -v gateway-data:/app/data   onlyoffice-gateway:v0.1.0

# 4. 检查日志
docker logs -f oo-gateway
```

**方式 C：二进制部署（开发/测试）**

```bash
make build
./bin/gateway -config gateway.yaml
# 输出: Gateway v0.1.0 listening on 0.0.0.0:18080
```


```bash
curl http://localhost:18080/api/v1/health
# 输出: {"status":"ok"}
```

## 阶段 3：业务服务接入

### 3.1 业务服务准备

业务服务需要一个 HTTP 端点接收 webhook + 一个前端页面嵌入编辑器。

**后端（任意语言）需要做的事：**

1. 保管好自己的 RSA 私钥（阶段 2.2 生成的 `crm-service-private.pem`）
2. 实现一个 webhook 接收端点：
   ```
   POST /internal/onlyoffice-callback
   Body: {
     "event": "document.saved",
     "document_id": "doc_xxx",
     "external_id": "...",
     "status": "ready",
     ...
   }
   ```
   收到后 → `GET /api/v1/documents/{document_id}` 从 Gateway 下载文件。

### 3.2 签 JWT 并上传文档

以 Node.js 为例（其他语言同理）：

```javascript
const jwt = require('jsonwebtoken');
const fs = require('fs');
const FormData = require('form-data');

const GATEWAY_URL = 'http://your-gateway-host:18080';
const PRIVATE_KEY = fs.readFileSync('crm-service-private.pem');

// 1. 签 JWT
const token = jwt.sign({
  service_id:  'crm-service',
  webhook_url: 'https://crm.your-company.com/internal/onlyoffice-callback',
  external_id: 'contract-2024-001',
  user: {
    id:   'user-123',
    name: '张三'
  },
  file_name:     '合同模板.docx',
  document_type: 'word',
  branding: {
    logo_url:    'https://crm.your-company.com/logo.png',
    language:    'zh-CN'
  }
}, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '60s' });

// 2. 上传文件
const form = new FormData();
form.append('file', fs.createReadStream('/path/to/合同模板.docx'));

const response = await fetch(`${GATEWAY_URL}/api/v1/documents`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    ...form.getHeaders()
  },
  body: form
});

const { document_id } = await response.json();
console.log('上传成功, document_id:', document_id);
// → 保存 document_id，前端需要用
```

### 3.3 验证上传成功

```bash
# 检查 Gateway 存储目录
ls -la data/storage/
# 应该看到一个以 doc_ 开头的目录，里面有 original.docx 和 meta.json
cat data/storage/doc_*/meta.json | python3 -m json.tool
```

## 阶段 4：前端集成

### 4.1 安装前端 SDK

```bash
npm install @zenmind/onlyoffice-editor
```

### 4.2 嵌入编辑器

```tsx
import { OnlyOfficeEditor } from "@zenmind/onlyoffice-editor";

function DocumentEditor({ documentId }: { documentId: string }) {
  const handleSaved = async (event: { document_id: string }) => {
    // 从 Gateway 下载编辑结果
    const resp = await fetch(
      `http://your-gateway-host:18080/api/v1/documents/${event.document_id}`,
      { headers: { Authorization: `Bearer ${getServiceJWT()}` } }
    );
    const blob = await resp.blob();
    // 保存到你的存储
    await uploadToStorage(blob);
  };

  return (
    <OnlyOfficeEditor
      documentId={documentId}
      gatewayUrl="http://your-gateway-host:18080"
      onReady={() => console.log("编辑器就绪")}
      onSaved={handleSaved}
      onError={(err) => console.error(err)}
      style={{ height: "80vh" }}
    />
  );
}
```

### 4.3 生成编辑器 token

`/edit` 端点需要 JWT token。业务服务后端需要提供一个获取 token 的接口：

```javascript
// 业务服务后端
app.get('/api/onlyoffice/edit-token', (req, res) => {
  const { documentId } = req.query;
  const token = jwt.sign({
    service_id:  'crm-service',
    document_id: documentId,
  }, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '30m' });
  res.json({ token });
});
```

前端调用：

```tsx
// 获取 editor token
const { token } = await fetch(`/api/onlyoffice/edit-token?documentId=${documentId}`).then(r => r.json());

// 传给组件（组件内部会拼 /edit?token=xxx）
<OnlyOfficeEditor
  token={token}      // ← 或者让组件内部通过 documentId 去请求
  gatewayUrl="http://your-gateway-host:18080"
  ...
/>
```

> 注意：当前 `OnlyOfficeEditor` 组件通过 `documentId` 直接构造 `/edit?document_id=xxx`。如果需要 token 模式，需更新组件 props 或由 Gateway 的 `/edit` 同时支持两种模式。**当前实现**：`/edit` 端点**要求 token**（阶段 2 的补齐结果），所以前端必须传 token。组件支持通过 `token` prop 直接传入，也支持自动从后端获取。

## 常见问题

### Q: Document Server 无法访问 Gateway 的 /download 和 /callback？

**原因**：Gateway 监听 `127.0.0.1`，Docker 容器无法访问宿主机的 localhost。

**解决**：Gateway 配置 `listen_addr: "0.0.0.0:18080"`。

### Q: 上传时提示 "invalid token"？

检查：JWT 算法是否为 RS256、私钥与 gateway.yaml 中的公钥是否配对、JWT 是否过期（`exp` 建议 60s）。

### Q: Webhook 域名白名单检查失败？

`allowed_webhook_domains` 检查的是 webhook URL 的**域名部分**（不含 scheme 和端口）。

例如 `https://crm.example.com:443/callback` 会提取 `crm.example.com` 与白名单匹配。

### Q: 文档 TTL 过期被清理了怎么办？

TTL 默认 8 小时。编辑期间每次 callback（status=1）都会自动续期。如果文档被清理后仍需编辑，重新上传即可。

### Q: 多个用户编辑同一文档能协同吗？

能。Gateway 在首次上传时生成 `document.key` 并持久化。所有后续打开同一 `document_id` 的请求复用同一个 key，Document Server 会将他们加入同一编辑会话。
