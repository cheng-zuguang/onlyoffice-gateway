## 阶段 0：生成 JWT Secret

Gateway 与 Document Server 共享同一个 JWT Secret，用于：
- Document Server 签发/验证编辑会话 token
- Gateway Admin 登录 session token
- Webhook HMAC 签名

### 生成方式

```bash
# 推荐：OpenSSL 随机生成 32 字节 hex
openssl rand -hex 32
# 输出示例: a1b2c3d4e5f6...（64 位 hex）

# 或者用 Python
python3 -c "import secrets; print(secrets.token_hex(32))"

# 或者用 Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**生产环境务必自定义此值**。Docker Compose 一键部署时，将此值填入 `.env` 的 `JWT_SECRET`，会自动同步到 Document Server 和 Gateway 两个容器。

## 阶段 1：Docker Compose 一键部署（推荐）

### 1.1 配置

```bash
cp .env.example .env
```

编辑 `.env`，最少设置：

```env
JWT_SECRET=a1b2c3d4...     # 阶段 0 生成的值
ADMIN_PASSWORD=强密码        # 管理端登录密码
```

完整配置项见 `.env.example`，均有合理默认值。

### 1.2 启动

```bash
docker compose up -d
```

启动三个容器：

| 容器 | 端口 | 说明 |
|---|---|---|
| `oo-document-server` | 18000 | ONLYOFFICE Document Server |
| `oo-gateway` | 18080 | Gateway API |
| `oo-admin-ui` | 18081 | 管理端界面（nginx） |

### 1.3 访问管理端

浏览器打开 `http://<host>:18081/admin/login`，用 `.env` 中设置的 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 登录。

### 1.4 查看状态

```bash
docker compose ps
docker compose logs -f gateway
```

## 阶段 2：手动部署（源码）

### 2.1 克隆项目

```bash
cd /opt
git clone <repo-url> onlyoffice-gateway
cd onlyoffice-gateway
```

### 2.2 配置

```bash
cp .env.example .env
```

编辑 `.env`：

```env
LISTEN_ADDR=0.0.0.0:18080
DOCUMENT_SERVER_URL=http://localhost:18000
JWT_SECRET=<阶段 0 生成的值>
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<强密码>
```

Gateway 启动时自动加载 `.env`，无需 `source` 或 `export`。

### 2.3 构建并启动

```bash
# 终端 1：启动 Gateway
make build
./bin/gateway

# 终端 2：启动管理端（开发模式）
make frontend-dev
```

管理端开发服务器运行在 `http://localhost:5173/admin/login`，通过 Vite proxy 自动将 `/admin/api` 转发到 Gateway 的 18080 端口。

### 2.4 单独部署 Document Server

如果不用 Docker Compose，手动启动 Document Server：

```bash
docker pull onlyoffice/documentserver:latest

docker run -d \
  --name onlyoffice-document-server \
  -p 18000:80 \
  -e JWT_SECRET=<与 Gateway 相同的值> \
  -v /opt/onlyoffice/DocumentServer/data:/var/www/onlyoffice/Data \
  onlyoffice/documentserver:latest
```

## 阶段 3：注册业务服务

打开管理端，登录后：

1. 点击 **Add Service**
2. 填写 **Service ID**（唯一标识，如 `crm-service`）
3. 粘贴业务服务的 **RSA 公钥**（PEM 格式）
4. 填写 **Webhook 域名白名单**（每行一个，或逗号/分号分隔）
5. 提交

新增的服务**立即生效**，无需重启 Gateway。后续可在管理端编辑或删除。

## 阶段 4：业务服务接入

### 4.1 生成 RSA 密钥对

```bash
openssl genpkey -algorithm RSA -out service-private.pem -pkeyopt rsa_keygen_bits:2048
openssl rsa -pubout -in service-private.pem -out service-public.pem
```

将 `service-public.pem` 的内容粘贴到管理端的 "RSA Public Key" 字段。私钥由业务服务保管，用于签名 JWT。

### 4.2 签 JWT 并上传文档

```javascript
const jwt = require('jsonwebtoken');
const fs = require('fs');
const FormData = require('form-data');

const GATEWAY_URL = 'http://your-gateway:18080';
const PRIVATE_KEY = fs.readFileSync('service-private.pem');

const token = jwt.sign({
  service_id:  'my-service',
  webhook_url: 'https://my-app.example.com/webhook',
  external_id: 'doc-001',
  file_name:   'document.docx',
  document_type: 'word',
}, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '60s' });

const form = new FormData();
form.append('file', fs.createReadStream('document.docx'));

const resp = await fetch(`${GATEWAY_URL}/api/v1/documents`, {
  method: 'POST',
  headers: { Authorization: `Bearer ${token}`, ...form.getHeaders() },
  body: form,
});
const { document_id } = await resp.json();
```

### 4.3 生成编辑 token

```javascript
app.get('/api/edit-token', (req, res) => {
  const token = jwt.sign({
    service_id:  'my-service',
    document_id: req.query.documentId,
  }, PRIVATE_KEY, { algorithm: 'RS256', expiresIn: '30m' });
  res.json({ token });
});
```

## 阶段 5：前端集成

### 5.1 安装 SDK

```bash
npm install @zenmind/onlyoffice-editor
```

### 5.2 嵌入编辑器

```tsx
import { OnlyOfficeEditor } from "@zenmind/onlyoffice-editor";

const { token } = await fetch(`/api/edit-token?documentId=${docId}`).then(r => r.json());

<OnlyOfficeEditor
  documentId={docId}
  gatewayUrl="http://gateway:18080"
  token={token}
  onSaved={async (e) => {
    const resp = await fetch(`${gatewayUrl}/api/v1/documents/${e.document_id}`, {
      headers: { Authorization: `Bearer ${await getServiceJWT()}` },
    });
    const blob = await resp.blob();
    await saveToStorage(blob);
  }}
  style={{ height: "80vh" }}
/>
```

## 常见问题

### Q: 如何生成 JWT_SECRET？

```bash
openssl rand -hex 32
```

Gateway 与 Document Server 必须使用相同的值。Docker Compose 部署时通过 `.env` 中的 `JWT_SECRET` 自动同步。

### Q: 管理端无法登录？

1. 确认 `.env` 中设置了 `ADMIN_PASSWORD`
2. 确认 Gateway 日志没有 `WARNING: ADMIN_PASSWORD not set`
3. Gateway 启动时会自动加载 `.env`，如果仍无密码，检查 `.env` 是否在正确目录

### Q: Document Server 无法访问 Gateway？

Gateway 需监听 `0.0.0.0`（容器默认配置），Docker Compose 已自动配置。

### Q: 上传时提示 "invalid token"？

检查：JWT 算法是否为 RS256、私钥与管理端注册的公钥是否配对、JWT 是否在有效期内。

### Q: Webhook 域名白名单检查失败？

检查的是 Webhook URL 的域名部分（不含 scheme 和端口）。在管理端编辑服务即可修改白名单。

### Q: 文档过期被清理？

TTL 默认 8 小时。编辑期间 callback（status=1）自动续期。可在 `.env` 中调整 `TTL_HOURS`。
